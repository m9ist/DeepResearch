# Deep Research

Локальный движок «глубокого исследования»: внешний оркестратор управляет
state-машиной (план → нарезка заданий → запуск подагентов → сбор → сводка →
решение продолжать), а одноразовые вызовы pi выступают воркерами. Истина —
файлы на диске, поэтому систему можно остановить в любой момент и продолжить.

## Language

**Run** (рус. «исследование», «прогон»):
Одно исследование от постановки задачи до финального отчёта. Живёт в отдельной
папке `runs/<дата>-<slug>/`. Носитель всего состояния.
_Avoid_: проект, сессия, задача.

**Orchestrator** (рус. «оркестратор»):
Долгоживущий внешний процесс (Python), владеющий state-машиной автономной фазы
Run'а: нарезает **Backlog** из **Spec**, запускает **Worker**'ов, собирает
**Finding**'и, пишет **Report**'ы. Интервью НЕ ведёт (это **Intake**). Не путать
с pi: оркестратор переживает рестарт, читая файлы Run'а.
_Avoid_: корневой агент, мастер, ведущий.

**Intake** (рус. «интейк», «приёмка»):
Интерактивная фаза в начале Run'а: сессия pi по prompt-template ведёт интервью с
человеком и в конце пишет **Spec** в папку Run'а. Отдельна от **Orchestrator**'а
— это разовый интерактивный pi, а не долгоживущий процесс.
_Avoid_: интервью-агент, бриф-сессия.

**Worker** (рус. «подагент»):
Одноразовый вызов pi (`pi -p`), выполняющий ровно одно задание из **Backlog**
и оставляющий после себя **Finding**. Не хранит состояние между запусками —
переживает рестарт не он, а его Finding на диске.
_Avoid_: subagent (англ.), исполнитель, краулер.

**Spec** (рус. «ТЗ»):
Документ с задачей исследования, сформированный оркестратором по итогам
интервью с человеком. Вход для нарезки **Backlog**.
_Avoid_: бриф, запрос, промпт.

**Backlog** (рус. «бэклог», «очередь заданий»):
Очередь заданий на исследование со статусами (ожидает / в работе / готово /
отброшено). Пополняется как при нарезке **Spec**, так и из ссылок «ещё
почитать», которые предлагают **Worker**'ы.
_Avoid_: список задач, todo, план (план — это разбиение Spec, не очередь).

**Round** (рус. «раунд», «волна»):
Тайм-боксированная волна работы. Внутри раунда **Worker**'ы идут непрерывно
(по слотам concurrency), вычерпывая ready-задания из **Backlog**, пока не
истечёт бюджет времени раунда или не опустеет очередь. Завершается
**Checkpoint**'ом. Незавершённые задания и новые ссылки «ещё почитать»
переходят в следующий раунд. Первый раунд по умолчанию короткий (5–10 мин),
чтобы первый результат был быстрым. «+N раундов» = провести ещё N волн.
_Avoid_: итерация, цикл, батч.

**Checkpoint** (рус. «чекпоинт», «точка останова»):
Момент в конце **Round**'а: оркестратор делает промежуточный синтез (**Report**),
переоценивает направление (re-plan, возможна корректировка) и решает,
продолжать ли. Естественная точка для паузы и человеческого вмешательства.
_Avoid_: барьер, стоп.

**Replan** (рус. «переплан», «дозагрузка вопросов/идей»):
Вмешательство человека на паузе: пользователь докидывает новые вопросы/идеи,
оркестратор синтезирует накопленные **Finding**'и + эти уточнения в НОВУЮ версию
плана (новую версию **Spec**/**Backlog**), с которой Run продолжается. Старые
версии не затираются — сохраняется историчность (см. ADR об иммутабельности).
_Avoid_: правка ТЗ, редактирование плана (затирание — это не Replan).

**Finding** (рус. «находка»):
`.md`-документ — результат одного действия **Worker**'а: куда сходил, что
вытащил, саммари, ссылки «ещё почитать», смежные темы.
_Avoid_: результат, отчёт (отчёт — это Report), заметка.

**Report** (рус. «сводка»):
Дайджест прогресса Run'а, собранный оркестратором из накопленных **Finding**'ов
(«прошло N минут — нашли то-то, в бэклоге ещё M ссылок»). Бывает периодическим
(по таймеру) и по требованию (кнопка «сформируй промежуточный отчёт»).
Терминального «финального» отчёта-как-конца-Run'а нет — Run всегда можно
продлить на ещё несколько **Round**'ов.
_Avoid_: отчёт (неоднозначно — используем «сводка»), статус, финал.

## Flagged ambiguities

- **«Оркестратор» vs «Intake».** В сырой постановке один «агент-оркестратор»
  и интервьюирует, и управляет. В дизайне это два разных существа:
  **Intake** (разовый интерактивный pi) и **Orchestrator** (долгоживущий
  Python-процесс). Слово «оркестратор» закреплено только за вторым.
- **«Отчёт».** Перегружено: и Finding, и Report по-русски «отчёт». Канон:
  результат одного Worker'а — **Finding** («находка»), дайджест по Run'у —
  **Report** («сводка»).

## Пример диалога

— Run завис на паузе, в бэклоге 30 заданий. Это незавершённый Round?
— Да. Ты нажал Pause посреди Round'а — in-flight Worker'ы убиты, их задания
  вернулись в ready. Текущий Round не закрыт, Checkpoint не наступил.
— Если нажму Resume — он доделает этот Round или начнёт синтез?
— Доделает: добивает ready-задания, пока не истечёт бюджет времени Round'а,
  и только тогда Checkpoint — синтез Report и решение, идти ли в следующий Round.
— А «+2 раунда» сейчас что сделает?
— Поднимет бюджет Round'ов на 2. Сам по себе паузу не снимает — нужен Resume.
— Откуда в бэклоге взялись 30, если в Spec я просил узко?
— Часть — нарезка Spec, остальное Worker'ы накидали в свои Finding'и как
  «ещё почитать», и оркестратор затянул их в Backlog на прошлых Checkpoint'ах.
